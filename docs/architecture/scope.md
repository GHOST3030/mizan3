# Mizan POS — وثيقة النطاق والتقسيم المعتمد

**الإصدار:** 1.0  
**التاريخ:** 2026-06-05  
**الحالة:** معتمد ✅

---

## 1. قرار المعمارية

النظام يُبنى كـ **Modular Monolith**:
- مشروع واحد (codebase واحد)
- قاعدة بيانات واحدة (PostgreSQL عبر Supabase)
- تقسيم منطقي داخلي إلى Modules — لا Microservices في هذه المرحلة

---

## 2. الـ Modules المعتمدة

| Module | المسؤولية |
|--------|-----------|
| `core` | الشركة، الفروع، الإعدادات العامة، العملات |
| `auth` | المستخدمون، الصلاحيات، JWT، الورديات |
| `products` | المنتجات، التصنيفات، الوحدات، العلامات التجارية |
| `inventory` | المخزون، حركات المخزون، الجرد |
| `sales` | المبيعات، فواتير البيع، المرتجعات |
| `purchases` | المشتريات، فواتير الشراء |
| `customers` | العملاء، الديون، نقاط الولاء |
| `suppliers` | الموردون |
| `finance` | الخزينة، المدفوعات، المصروفات |
| `reports` | التقارير والإحصائيات |

---

modules/<name>/

├── <name>.controller.js

└── <name>.validation.js

---

## 4. قواعد ثابتة لا تتغير

- كل الجداول تحتوي: `id (uuid)`, `created_at`, `updated_at`, `deleted_at`
- Soft delete في كل مكان — لا حذف حقيقي للبيانات الحساسة
- كل القيم المالية تُخزن كـ **integers** (هللات/فلوس) — لا float
- كل سجل مالي يحتوي: `currency_id` + `exchange_rate`
- البيانات المرتبطة بالفرع تحتوي دائماً: `branch_id`
- حركات المخزون تُسجَّل في `stock_movements` — لا تحديث مباشر للمخزون
- كل DB access يمر عبر **Prisma ORM** — لا raw SQL إلا للضرورة
