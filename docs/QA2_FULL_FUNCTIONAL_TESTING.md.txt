# PHASE QA-2 — Full Functional Testing & End-to-End Validation

IMPORTANT:

Do NOT add new features.

Do NOT perform refactoring unless required to fix a verified bug.

The goal is to functionally test the running system exactly like a real user.

All reports, findings, progress updates, and summaries must be written in Arabic.

---

## MISSION

Run the application and perform a complete functional audit.

This phase is NOT a code review.

This phase is NOT a static analysis.

This phase MUST verify actual runtime behavior.

Test the system exactly as a user would.

---

## ENVIRONMENT

1. Start Backend
2. Start Frontend
3. Verify database connection
4. Verify authentication works
5. Verify seed data exists

* No blank screen
