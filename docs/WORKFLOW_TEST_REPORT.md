## سير العمل A: عميل ← مبيعات ← خفض المخزون

### الخطوات
| الخطوة | الحالة | التفاصيل |
|--------|--------|----------|
| 1. الحصول على عميل موجود | ✅ نجاح | تم العثور على عميل بمجموعة VIP |
| 2. الحصول على منتج | ✅ نجاح | 7up Can 330ml (سعر التكلفة: 200، سعر البيع: 300) |
| 3. التحقق من المخزون قبل البيع | ✅ نجاح | الكمية: 71 وحدة |
| 4. فتح وردية | ❌ فشل | يتطلب `user_id` بصيغة UUID صحيحة |
| 5. إنشاء فاتورة مبيعات | ❌ فشل | يتطلب `shift_id` صحيح (وردة مفتوحة) |
| 6. التحقق من خفض المخزون | لم يكتمل | لم تتم عملية البيع |

**السبب:** لم نتمكن من فتح وردية جديدة لأن التحقق من `user_id` يرفض UUID صحيح. وتم استخدام وردية مفتوحة `accb8114-e735-4d79-896b-62d9dd7a1a48` ولكن API المبيعات ما زال يرفضها.

## المشاكل المكتشفة

1. **🔴 حرج:** فشل إنشاء المبيعات وفواتير المشتريات بشكل كامل بسبب خطأ التحقق من صحة `shift_id` و `user_id` كـ UUID. هذا يمنع سير العمل التجاري الأساسي.

2. **🔴 حرج:** خطأ في التحقق من `user_id` - قيمة UUID الصحيحة مثل `1ac5d710-eaa2-4f5e-95f1-b07086d29f65` يتم رفضها في بعض نقاط API كمُعَرَّف غير صالح ("Invalid uuid").

3. **🔴 متوسط:** ورديات العمل المفتوحة المتاحة حالياً هي لموظفين آخرين (مثل "حسين")، ولا يمكن إنشاء وردية للمستخدم الحالي (admin) بسبب مشكلة UUID.
